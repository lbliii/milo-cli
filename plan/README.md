# Planning Boundary

GitHub issues are Milo's canonical roadmap and work queue:

- [open issues](https://github.com/lbliii/milo-cli/issues) track active work;
- saga and epic issues carry dependencies and acceptance criteria; and
- priority and theme labels define ordering.

This directory is reserved for approved, durable implementation plans that
need more structure than an issue body. A plan must link its issue, name the
decision owner, scope and non-goals, dependencies, risks, proof, collateral,
and exit criteria. Plans are not a second backlog and do not override code,
tests, `AGENTS.md`, release metadata, or an issue's current decision.

Canonical product and contributor guidance remains in `README.md`, `docs/`,
`site/content/docs/`, `CONTRIBUTING.md`, and `SECURITY.md`.
