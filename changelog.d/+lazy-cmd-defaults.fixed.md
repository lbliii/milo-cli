Lazy commands now propagate function signature defaults to argparse. Schema defaults are JSON-safe, boolean schema defaults are respected, boolean `default=True` parameters use `--no-xxx` flags, schema `enum` values become argparse `choices`, and a new `display_result=False` option suppresses plain-format output while preserving `--format json`. `Group.lazy_command()` now supports `examples`, `confirm`, and `annotations` kwargs for parity with `CLI.lazy_command()`.
